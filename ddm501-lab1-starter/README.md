# Lab 1: First ML Product - Movie Rating Prediction API

## Overview

Movie Rating Prediction ML product using collaborative filtering (SVD), a FastAPI REST API, tests, and Docker.

Key features:
- Train a Surprise SVD collaborative filtering model on MovieLens 100K and save it to `models/svd_model.pkl`
- Serve predictions via REST:
  - `GET /health`
  - `POST /predict`
- Swagger UI available at `GET /docs`
- Dockerfile + docker-compose for containerized runs
- pytest suite covering happy path and validation failures

## Quick Start

### 1. Clone and Setup

```bash
cd ddm501-lab1-starter

# Create virtual environment
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows (PowerShell)

# Install dependencies
pip install -r requirements.txt
```

### 2. Train the Model

```bash
python scripts/train_model.py
```

This will:
- download MovieLens 100K via Surprise built-in downloader
- Train a Surprise SVD collaborative filtering model
- save the model to `models/svd_model.pkl`

### 3. Run the API

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Open Swagger UI:
- http://localhost:8000/docs

### 4. Test the API

```bash
# Health check
curl http://localhost:8000/health

# Predict
curl -X POST "http://localhost:8000/predict" \
  -H "Content-Type: application/json" \
  -d '{"user_id": "196", "movie_id": "242"}'
```

Notes:
- If the model file is missing, `/health` will be `"unhealthy"` and `/predict` returns HTTP 503.
- Predicted ratings are clamped to the range 1.0 to 5.0.
- Unknown `user_id` or `movie_id` are handled gracefully (Surprise still returns an estimate).

### 5. Run with Docker

```bash
docker compose build
docker compose up -d

# Verify health
curl http://localhost:8000/health

# Stop
docker compose down
```

## API Endpoints

| Method | Endpoint | Description |

## Running Tests

```bash
# Run all tests
pytest tests/ -v

# Run with coverage
pytest tests/ -v --cov=app --cov-report=html
```

## Configuration

Environment variables:
- `MODEL_PATH`: path to the pickled model (default: `models/svd_model.pkl`)
- `MODEL_VERSION`: version string returned in prediction responses (default: `1.0.0`)

Example:
```bash
export MODEL_PATH=/absolute/path/to/svd_model.pkl
uvicorn app.main:app --host 0.0.0.0 --port 8000
```
